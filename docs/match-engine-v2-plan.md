# VCFM 比赛引擎 v2 设计方案（空间模拟 / FM·FMM 风格）

> 目标：把"数值掷骰 + 事后编舞"的表演层，换成"真实空间模拟的俯视投影"。
> 约束：纯前端 JS、手机浏览器可跑、**不破坏现有存档/统计/赛后报告/联赛积分**。

---

## 1. 现状与差距根源

### 现在的数据流（v1）

```
match.js  逐分钟掷骰子(tryAttack/tryCard/tryInjury) → event 流(已定结果)
   │  onEvent(ev, snap)
   ▼
matchview.js  当"导演",事后编排球员走位去"演"这些既定事件
```

**差距根源**：结果先被概率决定，画面只是演出。球员不做决策，所以：
- 每场跑位雷同，没有战术因果
- 抢断/传球/跑位不是"涌现"的，是被剧本安排的
- 无论怎么精修走位，都到不了 FM 那种"活着"的感觉

### FM / FMM 的本质

FM 的 2D 不是表演层，是**连续空间模拟的俯视镜头**。3D 版只是换镜头，跑的是同一套模拟：
- 每个球员是一个 agent，按固定频率（~5–10Hz）基于**当前真实场面**做决策
- 球有独立物理（位置、速度、地面摩擦、腾空）
- 进球、抢断、越位全部**从模拟中涌现**，没有预设结果

**结论**：要还原 FM 体验，唯一的路是把比赛核心换成空间模拟。这是本方案的全部内容。

## 2. 目标架构（v2）

```
                    ┌─────────────────────────────────────┐
                    │  SimEngine（新增，纯逻辑，无 DOM）    │
                    │  以固定步长 dt=0.1s 推进：            │
                    │   · 22 个 player agent 各自决策        │
                    │   · ball 物理                          │
                    │   · 裁判规则(越位/犯规/出界/进球)      │
                    │   · 每步产出 SimState 快照             │
                    └───────────┬─────────────────┬─────────┘
                                │                 │
                 每模拟分钟聚合  │                 │ 每 tick 快照
                                ▼                 ▼
                    ┌───────────────────┐   ┌──────────────────────┐
                    │ match.js 适配层     │   │ matchview.js 渲染层   │
                    │ 把涌现事件翻译成    │   │ 直接画 SimState:      │
                    │ 现有 event 结构     │   │ 球员坐标/球/朝向       │
                    │ (goal/card/injury) │   │ (复用今天加的惯性等)   │
                    │ → 喂给现有统计/报告 │   │ 不再"导演",只"显示"    │
                    └───────────────────┘   └──────────────────────┘
```

核心原则：**SimEngine 是唯一真相来源。** match.js 从"产生结果"退化为"翻译结果 + 记账"，matchview 从"导演"退化为"显示器"。这样现有的赛后报告、评分、积分榜、存档全部不用动接口。

## 3. SimEngine 详细设计

### 3.1 坐标与时间
- 场地统一用现有的 `0..100 × 0..100` 百分比坐标（matchview 已经是这套，零转换）
- 固定步长 `dt = 0.1s`（10Hz）。90 分钟真实比赛 = 54000 步，但我们按**时间加速比**跑：直播模式实时投影，快速模拟一次性跑完
- 每步：先所有 agent 决策 → 再积分位置 → 再裁判判定 → 产出快照

### 3.2 球物理（简单够用）
```
ball: { x, y, vx, vy, z, vz, owner }
- 地面滚动：每步 v *= friction(0.96/步)
- 传球/射门：给定初速 + 目标，抛物线(z)或地面滚
- owner==null 时可被最近的、在控球半径内的球员"接管"
- 接管概率 = f(球员 first_touch/技术, 相对速度, 距离)
```

### 3.3 球员 Agent 决策模型（核心中的核心）

每个球员每 tick 按状态决策。分**持球者 / 无球进攻方 / 防守方**三类：

**持球者**（评分选最优动作）：
```
候选动作打分，取最高：
  · 射门   = f(离球门距离/角度, shooting/finishing, 面前防守压力)
  · 传球   = 对每个队友算传球价值 f(推进距离, 接球点安全度,
             传球线是否被拦, passing/vision) → 选最优接球人
  · 盘带   = f(前方空间, dribbling/pace, 最近防守者距离)
  · 护球   = 高压下的保底
决策频率不必每 tick，0.2~0.4s 一次，中间执行上次决策
```

**无球进攻方**（跑位）：
```
· 拉开空间 / 前插跑动（找防线身后或肋部空当）
· 保持接应角度（给持球人可传的三角）
· 越位线自律（已有 _clampTargetOffside 逻辑可迁移）
```

**防守方**：
```
· 最近人上抢持球者（closing down）
· 其余人盯人 or 区域协防（看战术 marking 设置）
· 保持防线整体性（今天加的分离力 + 防线 Y 对齐）
· 抢断判定 = f(tackling/marking, 位置, 时机) → 成功则 owner 易主
```

**决策全部读现有球员属性**（`pace/passing/shooting/dribbling/tackling/vision/positioning/...`），所以强队弱队、不同球员风格会自然拉开差距——这就是 FM 感的来源。

### 3.4 战术接入
现有 `tactics`（阵型、`defensiveLine`、压迫、风格）直接作为 agent 参数：
- 阵型 slots → 每个球员的"基准位置"（base position），跑位围绕它浮动
- `defensiveLine` → 防线整体 Y 高度 + 越位陷阱倾向
- 压迫强度 → 防守方 closing down 的触发距离与速度
- 进攻风格 → 传球倾向（短传渗透 vs 长传直塞）

**这一步让"改战术真的改变比赛"**，而不是像现在只改概率权重。

### 3.5 裁判规则
- **出界**：球出场 → 界外球/角球/球门球，owner 给对应方
- **越位**：传球瞬间记录接球人与倒数第二名防守者的 Y → 判定
- **犯规/卡牌**：抢断失败 + 接触 → 按 `tackling` 反比 + 战术凶狠度掷犯规 → 复用现有 `discipline.js` 的黄红牌逻辑
- **进球**：球整体越过门线 → 进球事件
- **伤病**：低频随机 + 高强度对抗触发 → 复用现有 `tryInjury` 的结果结构

## 4. 与现有系统的衔接（关键，保证不破坏存档）

### 4.1 事件翻译层
SimEngine 涌现出关键事件时，翻译成**现有 event 结构**（`{minute, type, text, extra}`），继续走 `pushEv` / `onEvent`。这样：
- ✅ 赛后报告 `buildReport` / 叙事 `buildMatchNarrative` 不用改
- ✅ 球员评分 `applyMatchRatings` 不用改（甚至能更准，因为有真实数据）
- ✅ 积分榜 `applyResult`、存档结构不用改

### 4.2 统计来源升级
现在 xG/控球/射门是概率累积。v2 里这些**从模拟真实产出**：
- 控球% = 累计 owner 时间
- 射门/xG = 真实射门位置和角度算 xG
- 传球网络/热区 = 真实轨迹（matchview 已有热区/传球网络 UI，直接喂真数据）

### 4.3 三种模拟模式
- **直播**：SimEngine 实时步进，matchview 每 tick 画 → FMM 式观赛
- **快速模拟**：SimEngine 无渲染全速跑完 → 只出结果和事件
- **一键完赛/后台比赛**：同快速模拟（联赛里其他场次用这个，性能要够轻）

## 5. 性能预算（手机浏览器）

- 22 agent × 10Hz，每 agent 决策是几十次距离/打分计算 → 单场快速模拟约 54000 步
- 优化：无球员决策降到 3~5Hz、决策打分只对"相关"队友/对手（近邻），空间用简单网格加速
- 联赛一轮十几场后台赛：用**更低精度**（可选 1Hz 或退回概率模拟出结果）避免卡顿
- 目标：中端手机单场快速模拟 < 200ms，直播稳定 30fps

## 6. 落地阶段（增量、每步可玩可回退）

> 用 feature flag 切换 v1/v2，随时对比、可回退。

**P0 — 骨架（不影响现有玩法）**
- 新建 `js/sim/engine.js`：球物理 + 空推进循环 + SimState 快照结构
- 先只放 22 个球员做"阵型站位 + 跟球"，无决策，验证渲染管线接通

**P1 — 持球与传球**
- 持球者决策（传/带/射）+ 接球 + 球物理
- 无球方基础接应跑位
- 能自然打进球（涌现第一个进球时是里程碑）

**P2 — 防守与抢断**
- closing down + 抢断判定 + 盯人/区域
- 犯规/卡牌接入 discipline.js

**P3 — 裁判规则完整**
- 越位、出界、角球、任意球、点球

**P4 — 战术深度 + 平衡**
- tactics 全参数接入
- 大量模拟跑分，校准到"强队约该赢、比分分布合理"（对齐现有联赛平衡）

**P5 — 事件翻译层 + 统计升级 + 性能优化**
- 接现有报告/评分/积分
- 后台赛性能分级

**P6 — 切换与清理**
- flag 默认切 v2，保留 v1 作为"极速"选项或删除
